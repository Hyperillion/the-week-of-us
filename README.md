# The Week of Us - 每周双向默契互评小屋 🏡

> **“我们不是为了争个对错，而是为了在彼此的差异中，找到最舒适的距离。”**

`The Week of Us` 是一款专为伴侣设计的高颜值、有温度的双向默契互评应用。它帮助伴侣通过每周“自评”与“TA评”的趣味问卷，在沟通、尊重边界、说到做到等日常表现中寻找共识，并生成深度的关系洞察雷达图与悄悄话留言板，建立健康、稳固、持续成长的亲密关系。

---

## ✨ 核心特色功能

1. **双向心智评分 (Double Evaluation Rubric)** 📝
   * 每周进行 **10 项相处行为维度**（尊重意愿、坦诚相待、说到做到等）和 **8 项综合体感维度**（开心幸福、安全信任、亲密连接等）的打分。
   * **动态标准说明**：滑块下方根据具体指标与分数实时渲染具体的场景化描述（如“1分：完全忽视TA的底线” / “5分：极度尊重TA的边界”），打分更客观。

2. **默契报告解锁机制 (Dashboard & Report)** 🔓
   * **独立填写**：当周需要双方独立提交评分，在双方均提交前，报告处于“锁住”状态，防止盲目跟从评分。
   * **关系综合得分**：利用加权算法（TA评 35% + 自评 25% + 综合体感 40%）输出百分制关系得分，并给出科学的等级诊断（如：灵魂共振、磨合进阶）。

3. **数据可视化 (Data Visualization)** 📊
   * **相处表现雷达图**：纯数学驱动的动态 SVG 雷达图，重合度越高说明彼此心智越同频。
   * **往期得分走势图**：动态 SVG 折线图并带渐变填充，记录长期的情感稳定曲线。

4. **冲突修复与加减分机制** 🟢🟡🔴
   * 引入“冲突评估”，吵架逃避冷暴力会发出预警（-15分），但如果双方选择直面解决（+5分）或达成完美共赢（+10分）则会获得满分肯定，鼓励积极解决问题。

5. **彼此的三句话碎碎念 ✉️**
   * 每周留下三句话：这周最幸福的事 💖、这周希望被改善的事 💡、下周愿意为关系做出的具体行动 🏃‍♂️。提交后在报告解锁时供彼此查阅。

6. **轻量化高级气泡提示 (Premium Toast)** 💎
   * 采用无厚重描边的全包裹微透玻璃拟态 Toast 提示，带有一丝微妙色泽的极细边框（Apple & Vercel 风格），提示更具质感。

## 🛠️ 技术栈与架构

### 前端 (Vite React SPA)
* **核心框架**：React 19 + TypeScript (TS)
* **工程构建**：Vite 8
* **样式设计**：原生 Vanilla CSS (内置暗黑模式 Dark Theme / 高级玻璃拟态 / 响应式布局 / 顺滑的弹窗渐入过渡)
* **语法规范**：Oxlint (超快速零配置 linter 校验)

### 后端 (Node.js Serverless Functions)
* **接口服务**：Vercel Serverless Functions (目录在 `/api/*`)
* **本地开发服务**：基于 `dev-server.js`，本地启动 Express 形式的中间件路由和 JSON 数据库。
* **数据持久化**：
  * **本地**：自动写入本地缓存文件 `/tmp/the_week_of_us_dev.json`。
  * **生产**：支持 Vercel KV (Upstash Redis) 极速缓存，实现零门槛云端部署。

## 🚀 快速启动指南

### 1. 安装依赖

首先，请确保你已经安装了 [Node.js](https://nodejs.org/) (推荐 v18+)。在项目根目录下执行安装命令：

```bash
npm install
```

### 2. 本地开发启动

执行启动脚本，系统将**同时启动前端开发服务器与本地后端服务**：

```bash
npm run dev
```

启动成功后：
* **前端界面**：在浏览器中打开 `http://localhost:5173/`。
* **本地数据**：所有的交互评分会自动写入你电脑的临时目录：`/tmp/the_week_of_us_dev.json`，即使刷新或重启服务数据也不会丢失。

### 3. 代码校验与构建

* **代码质量检测 (Linter)**：
  ```bash
  npm run lint
  ```
* **生产编译打包**：
  ```bash
  npm run build
  ```

## ☁️ 部署到 Vercel (零配置云端上线)

该项目使用标准 Serverless 架构，可以完美契合 Vercel 进行免费部署：

1. 安装 Vercel CLI 并登录：
   ```bash
   npm install -g vercel
   vercel login
   ```
2. 在项目根目录下执行部署指令即可自动识别后端逻辑：
   ```bash
   vercel
   ```
3. *(可选)* 如果你需要支持持久化云端数据库，请在 Vercel 仪表盘中为本项目关联一个 **Vercel KV (Redis)**，项目代码会自动读取环境变量 `KV_REST_API_URL` 和 `KV_REST_API_TOKEN` 并切换到云数据库存储。
